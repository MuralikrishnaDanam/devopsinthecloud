Setup a dynamic configuration management database

Defining a configuration management database

  • The jenkins.template template configures the create_config_domain.rb file in /usr/share/tomcat6/scripts/aws/ on the Jenkins instance.
  • To test this, you can go to Jenkins and launch the TargetEnvironment job.
  • A shell command within the TargetEnvironment job calls the create_config_domain.rb script. This script creates a SimpleDB domain and pushes configuration information into this persistent domain.
  • When the Deployment job in Jenkins is called, it executes the deploy.rb script. In this script, it reads the configured values from the SimpleDB domain that were defined when running the TargetEnvironment job.
  • When terminating a TargetEnvironment stack, it will delete the configuration domains that were previously setup.
  • You can run a job called CentralizedConfiguration to get the current configured values in the SimpleDB configuration domain.
  • OPTIONAL: Open the SimpleDB Tool to view the configuration that has been applied to SimpleDB.
