2.3 Document Test Scenarios

Learning Objectives

By the end of this lesson you will be able to -

  • Document the steps for deploying a software package.

Assumptions

  • None

High-Level Steps: Infrastructure

  • Verify that MySQL is installed and running.
  • Verify that JDK is installed and running.
  • Verify that Ant is installed and running.
  • Verify that Tomcat6 is installed and running.
  • Verify that Git is installed and running.
  • Verify that Subversion is installed and running.
  • Verify that Maven is installed and running.
  • Verify that Linux is installed and running.

High-Level Steps: Application Code

  • Verify that user gets invalid username/password message when entering incorrect username or password.

High-Level Steps: Deployment (Configuration and Data)

  • Verify that you can retrieve records from database.

EXERCISE: Document additional test scenarios

  • Document additional test scenarios for infrastructure provisioning and software application functionality. Document these steps in text file and upload your answers to your cd29artifacts S3 bucket (where 00 is your student id).
